# v2.0.4 - Gate 28 Runtime Readiness

**Date:** 2026-06-12
**Type:** Patch
**Runtime namespace:** `v2.0.4-gate28-runtime`
**Canon version:** `v2`

v2.0.4 closes Gate 28. It keeps the immutable v2 canon artifacts unchanged and
ships runtime readiness fixes around loading ownership, version truth,
Documents, root environment loading, and route readiness.

## Changes

- Promoted public runtime metadata to `liveVersion: v2.0.4` while keeping
  runtime UI split by `liveVersion` and `devVersion`.
- Reused the shared Limni loading surface for startup and route transitions,
  including the progress bar.
- Removed visible App Truth route blockers from user pages while preserving
  route truth metadata for diagnostics.
- Restored repo-root environment loading for auth and database access when the
  app server starts from `app/`.
- Reduced Data route boot fanout so selected reports load first and other
  reports load only when requested.
- Simplified Data controls by removing static explanatory labels and aligned
  Sentiment summary cards with the other Data sections.
- Reworked Documents into a modular release index with version tabs, newest
  history first, and click-to-enlarge evidence screenshots.

## Verification

- TypeScript project check passed.
- Release version consistency test passed.
- Canon shard and closed-week delta tests passed without changing
  `releases/v2/canon/*.json`.
- Production build passed.
- Playwright checked the app route set, including Data, Performance,
  Documents, Status, Automation, Research, News, Agents, Matrix, Flagship, and
  Antikythera. Dynamic account detail routes were skipped because no connected
  account records were present.
