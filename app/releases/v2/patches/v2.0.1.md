# v2.0.1 - Week Rollover Cache Readiness Patch

**Date:** 2026-05-31
**Type:** Patch
**Canon reuse:** Yes - `canonVersion: v2` unchanged, no canon re-download
**Cache invalidation:** Yes - `cacheNamespace: v2.0.1` forces client-side cache refresh

## What Changed

- Week rollover cache readiness fix so current-week slices no longer mask historical week data.
- Closed-week refresh now writes execution-anchor weekly returns in addition to canonical weekly returns, so newly closed ADR Grid weeks do not render as empty after rollover.
- Current-week Basket now uses the updated inline hierarchy-style renderer instead of the legacy `show detail` row list.
- Version discipline formalization: five-field manifest, `cacheNamespace` / `canonVersion` separation, and `preparedAt` / `releasedAt` distinction.
- Preload UI copy now uses exact `appVersion`.
- The blocking preload gate now covers only version/canon readiness. Strategy payloads and current-week refreshes start after canon but do not keep the full app on the loading screen.
- Badge popover shows patch date and v2 release line date.

## Why

v2.0.0 shipped with runtime-version and canon-version identity conflated. Codex pushed an initial week rollover fix as code, but without bumping any version field, browsers had no cache invalidation signal. The fix was present in production, but clients could keep restoring stale v2 state.

v2.0.1 separates the runtime version (`appVersion: v2.0.1`) from the canon artifact identity (`canonVersion: v2`), so cache invalidation happens without re-downloading the v2 canon bundle.

The patch also removes legacy pre-v2 strategy repair/live-week work from the blocking preload path. Historical all-variant preload is now the canon layer; runtime strategy and current-week refresh work runs after the app opens.

During browser verification, the previous-week symptom persisted because the rollover refresh had produced canonical weekly return rows but not execution-anchor rows. ADR Grid strategy artifacts read execution-anchor rows, so the closed May 25 week had signals but no price rows to execute against. The refresh job now derives both anchor families from canonical hourly bars and checks coverage separately for canonical and execution anchors.

## How To Verify

1. Clear browser caches once: `localStorage`, IndexedDB Limni stores, and browser Cache Storage.
2. Reload the app.
3. Confirm the badge popover reports `appVersion: v2.0.1`.
4. Verify May 25 historical week renders correctly.
5. Verify canon bundle was not re-downloaded when an existing `canonVersion: v2` bundle is already present.
6. Confirm `/api/performance/strategy-artifacts/status` reports all 12 strategy artifacts ready with no stale weeks.
7. Confirm current-week Basket uses the updated inline row style, not the legacy `show detail` list.
8. Confirm cached reload opens after restoring canon, without cycling through strategy/current-week phases in the full-screen loader.
