# v2.0.5 - Gate 29 Performance Data Correctness

Date: 2026-06-12

Type: Patch

Runtime namespace: `v2.0.5-gate29-performance-data-correctness`

Canon version: `v2`

## Summary

Gate 29 fixes visible Performance data correctness and UI parity issues without
regenerating or mutating the immutable v2 canon artifacts. Current and stored
weeks now use the same Basket hierarchy path, stored Simulation views preserve
Equity/Balance visibility, Summary card tone follows signed return, COT pending
copy no longer uses the old sentiment-only language, and chart colours remain
visible in both light and dark mode.

This is not a strategy optimization release. ADR Grid app-vs-indicator parity,
missing current-week fill/gain behaviour, and weekly source rollover logic stay
queued for the next gate.

## Changes

- Shared the selected trade-row Basket hierarchy across current, stored, and
  all-time Performance views.
- Added planned ADR grid rows beside filled rows so Tandem ADR Grid Basket views
  preserve 144 total grids and 36 grids per portfolio without counting planned
  rows as fills.
- Preserved selected-ledger Simulation Equity, Balance, and Total paths for
  stored weeks.
- Changed Summary portfolio card tone to follow signed return instead of
  win-rate gates.
- Replaced old sentiment-only COT warning copy with awaiting-CFTC wording.
- Remapped legacy white chart series colours to theme-safe accent colours for
  light and dark mode.
- Added focused regression coverage for planned ADR grid rows in selected
  ledgers.
