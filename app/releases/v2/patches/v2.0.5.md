# v2.0.5 - Gates 29-32 Performance Readiness

Date: 2026-06-12

Type: Patch

Runtime namespace: `v2.0.5-gate29-performance-data-correctness`

Canon version: `v2`

## Summary

v2.0.5 is the cumulative Gate 29-32 readiness patch. It fixes visible
Performance data correctness, Friday rollover/source readiness, Basket expansion
parity, and Performance speed/cache behavior without regenerating or mutating
the immutable v2 canon artifacts.

This is not a strategy optimization release. ADR Grid app-vs-indicator parity,
weekly-hold signal qualification, and strategy model optimization stay queued
until current numbers are trusted.

## Changes

- Shared the selected trade-row Basket hierarchy across current, stored, and
  all-time Performance views.
- Added planned ADR grid rows beside filled rows so Tandem ADR Grid Basket views
  preserve 144 total grids and 36 grids per portfolio without counting planned
  rows as fills.
- Preserved selected-ledger Simulation Equity, Balance, and Total paths for
  stored weeks.
- Changed Summary portfolio card tone to follow signed return instead of
  win-rate gates.
- Replaced old sentiment-only COT warning copy with awaiting-CFTC wording.
- Remapped legacy white chart series colours to theme-safe accent colours for
  light and dark mode.
- Added focused regression coverage for planned ADR grid rows in selected
  ledgers.
- Consolidated Friday rollover/source readiness so the just-closed week freezes
  through source/certification evidence while the new display week appears in
  Data and Performance after the readiness cutoff.
- Reused Basket expansion behavior across Weekly Hold and ADR Grid, flattened
  single-trade Weekly Hold symbol rows, added direction/source context to grid
  headers, and reset expansion state on week/view changes.
- Memoized heavy Performance view projections so Summary, Simulation, and Basket
  tab switches do not recompute the full selected runtime every click.
- Reused the existing persistent strategy kernel payload cache on valid repeat
  Performance visits.
- Aligned the client router cache with the hourly cron cadence.
