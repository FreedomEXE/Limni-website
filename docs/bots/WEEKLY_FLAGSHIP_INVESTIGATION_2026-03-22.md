## 2. Bias-Gate Overlay Universal V1

Source:

- [strategy-comparison-reduce-as-skip.json](c:/Users/User/Documents/GitHub/limni-website/reports/bias-gate/strategy-comparison-reduce-as-skip.json)
- generated by [backtest-strategy-gate-comparison.ts](c:/Users/User/Documents/GitHub/limni-website/scripts/backtest-strategy-gate-comparison.ts)

What this report actually is:

- an overlay study on weekly snapshot trade returns
- applies BTC/ETH gate decisions as skip/reduce logic
- compares baseline vs gated performance
- uses `performance_snapshots` / derived `tiered` inputs
- **not** a canonical persisted `strategy_backtest_runs` flagship backtest

Critical math issue in the generator:

- `computeMetricsFromState()` sums weekly returns:
  - `baselineTotal = state.weeklyBaseline.reduce(...)`
  - `gatedTotal = state.weeklyGated.reduce(...)`
- drawdown is computed from the compounded weekly equity curve

So the gate report mixes:

- additive total return
- compounded drawdown

That makes it directionally useful, but not a final flagship audit source.

## Why This Matters

The repo currently mixes:

- legacy carry-based Universal DB runs
- overlay gate comparison studies
- normalized DB-backed flagship runs

If we do not separate these, we can accidentally compare:

- a 6-week carry model with floating losses
- against an 8-week trade-skip overlay
- against normalized 8-week DB-backed strategies

That is not apples-to-apples.

## Underlying Model Inputs

The foundational COT-derived inputs are:

- `dealer`

But the strategy families do **not** use only those three raw inputs:

- `v1`: `antikythera`, `blended`, `dealer`, `commercial`, `sentiment`
- `v2`: `dealer`, `sentiment`, `antikythera_v2`
- `v3`: `antikythera_v3`, `dealer`, `commercial`, `sentiment`

So the flagship investigation must treat “Universal/Tiered” as strategy families layered on top of those core inputs, not as simple three-model composites.

## Practical Read Right Now

### What is solid

- The gate overlay is strong evidence that gating materially improves the Universal family risk/return shape.
- Universal V1 gated is promising enough to deserve promotion testing.
- The legacy Universal V1 carry run clearly has real alpha, but also obvious floating-risk baggage.

### What is not yet promotable

- Any gate-overlay result as a final flagship without a dedicated rerun.
- Any weekly flagship claim that mixes additive overlay totals with canonical DB runs.

## Recommendation

Do **not** shelve Universal V1.

But also do **not** crown it yet from the overlay report alone.

The correct next step is:

1. Build a dedicated 8-week **gated Universal V1 rerun**
2. Persist it into `strategy_backtest_runs`
3. Audit that run from raw weekly rows
4. Then compare against the current normalized weekly candidate

### Promotion rule

If the dedicated 8-week gated Universal V1 rerun still shows:

- `> 100%` compounded return
- low week-close drawdown
- acceptable trade count
- and no hidden carry/floating pathology comparable to the 6-week legacy run

then it should replace the current placeholder weekly flagship.
